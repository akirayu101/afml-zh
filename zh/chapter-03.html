第一部分：数据分析

标签化

3.1 动机

在第2章中，我们讨论了如何生成矩阵 \(X\) 从非结构化数据集中提取的金融特征。无监督学习算法可以从该矩阵中学习数据模式 \(X\)，例如是否存在层次聚类结构。另一方面，监督学习算法要求 \(X\) 中的各行与标签或数值的数组相关联 \(y\)，以便在未见特征样本上预测这些标签或数值。本章将讨论金融数据的标签化方法。

3.2 固定时间窗口法

就金融领域而言，几乎所有ML论文均采用固定时间窗口法对观测值进行标签化。该方法描述如下。考虑一个特征矩阵 \(X\) 与 \(I\) 行， \(\{X_i\}_{i=1,\ldots,I}\)，从索引为 \(t=1,\ldots,T\)，其中 \(I\le T\)的某些 bar 中采样得到。第2章2.5节讨论了生成特征集合的采样方法 \(\{X_i\}_{i=1,\ldots,I}\)。某一观测值 \(X_i\) 被赋予标签 \(y_i\in\{-1,0,1\}\):

\[y_i=\begin{cases}-1, & r_{t_{i,0},t_{i,0}+h}<-\tau,\\0, & |r_{t_{i,0},t_{i,0}+h}|\le\tau,\\1, & r_{t_{i,0},t_{i,0}+h}>\tau,\end{cases}\]

其中 \(\tau\) 为预先设定的常数阈值， \(t_{i,0}\) 为观测值 \(X_i\) 发生后紧接着的 bar 的索引， \(t_{i,0}+h\) 为第 \(h\)个 bar 之后的索引， \(t_{i,0}\)，且 \(r_{t_{i,0},t_{i,0}+h}\) 为在 bar 时间窗口内的价格收益率 \(h\):

\[r_{t_{i,0},t_{i,0}+h}=\dfrac{p_{t_{i,0}+h}}{p_{t_{i,0}}}-1\]

由于文献中几乎总是使用时间条（time bar）， \(h\) 意味着固定的时间窗口。参考文献部分列出了多篇ML研究，其中 Dixon 等人 [2016] 是这种标签化方法的近期示例。尽管该方法颇为流行，但在大多数情况下应避免使用，原因有以下几点。第一，如第 2 章所述，时间条/time bar 的统计特性不佳。第二，无论观测到的波动率如何，同一阈值 \(\tau\) 均被固定应用。假设 \(\tau=1E-2\)，有时我们将某个观测值标记为 \(y_i=1\) ，对应的实现 bar 波动率为 \(\sigma_{t_{i,0}}=1E-4\) （例如，夜盘时段），有时则为 \(\sigma_{t_{i,0}}=1E-2\) （例如，开盘附近）。绝大多数标签将为 \(0\)，即便收益率 \(r_{t_{i,0},t_{i,0}+h}\) 具有可预测性且统计上显著。

换言之，依据时间条/time bar 上的固定阈值对观测值进行标签化，是一种极为常见的错误。以下是两种更优的替代方案。第一，采用动态变化的阈值 \(\sigma_{t_{i,0}}\)进行标签化，该阈值通过滚动指数加权收益率标准差估计得到。第二，使用成交量条或成交额条，因为其波动率更接近常数（同方差性）。然而，即便采用上述两项改进，也仍然忽略了固定时间窗口法的一个核心缺陷：价格所经历的路径。每种投资策略都设有止损限额，无论是组合经理自行设定、风险部门强制执行，还是由追加保证金通知触发。基于那些本会被交易所强制平仓的持仓来构建盈利策略，根本不切实际。几乎没有任何文献在对观测值进行标签化时将这一点纳入考量，这本身就说明了当前投资研究文献的现状。

3.4 三重障碍法

在此，我将介绍一种文献中尚未见到的替代标签化方法。如果您是投资从业者，我相信您会认同这种方法更具实践意义。我将其称为三重障碍法，因为它根据三条障碍线中最先被触及的那条来对观测值进行标签化。首先，我们设置两条水平障碍线和一条垂直障碍线。两条水平障碍线由止盈和止损限额定义，它们是估计波动率（无论是已实现波动率还是隐含波动率）的动态函数。第三条障碍线以持仓后经过的 bar 数量来定义（即到期限制）。若上方障碍线首先被触及，则将该观测值标记为 \(1\)。若下方障碍线首先被触及，则将该观测值标记为 \(-1\)。若垂直障碍线首先被触及，则有两种处理方式：取收益率的符号，或取 \(0\).

就在限定范围内实现盈亏而言，我个人倾向于前者，但您应探索 \(0\) 在您的特定问题中是否效果更佳。您可能已注意到，三重障碍法具有路径依赖性。为了对观测值进行标签化，我们必须考虑跨越 \([t_{i,0},t_{i,0}+h]\)，其中 \(h\) 定义垂直障碍线（到期限制）。我们将 \(t_{i,1}\) 记为首次触及障碍线的时刻，与所观测特征相关联的收益率为 \(r_{t_{i,0},t_{i,1}}\)。为表述清晰， \(t_{i,1}\le t_{i,0}+h\) 且水平障碍线不必对称。

代码片段 3.2 实现了三重障碍法。该函数接收四个参数：

    • ptSl[1]：乘以 trgt 以设定下方障碍线宽度的系数。若为 0，则不设下方障碍线。
  • molecule：包含将由单个线程处理的事件索引子集的列表。其用途将在本章后续内容中进一步阐明。

该函数的输出为一个 pandas 数据框，其中包含每个障碍被触及时的时间戳（如有）。如前述说明所示，该方法考虑了三个障碍中任意一个可能被禁用的情况。我们以三元组 [pt,sl,t1] 表示一种障碍配置，其中 0 表示该障碍未启用，1 表示该障碍已启用。共有以下八种可能的配置：

  • 三种实用配置：
    • [1,1,1]：这是标准设置，定义了三个障碍退出条件。我们希望实现盈利，但对亏损有最大容忍度，并设有持仓期限。
    • [0,1,1]：在该设置中，我们希望在持有若干 bar 后退出，除非被止损触发而提前离场。
    • [1,1,0]：在此设置中，我们希望在未被止损的前提下尽可能止盈。这一设置略显不切实际，因为我们愿意无限期持有仓位直至目标达成。
  • 三种现实意义较弱的配置：
    • [0,0,1]：等价于固定时间窗口法。该配置在应用于成交量条、成交额条或信息驱动 bar 时仍有一定实用价值，尤其是当多个预测在持仓期内持续更新时。
    • [1,0,1]：持有仓位直至盈利或超过最大持仓期，对中途的未实现亏损不加以限制。
    • [1,0,0]：持有仓位直至盈利为止，这可能意味着在亏损仓位上被套数年之久。
  • 两种不合逻辑的配置：
    • [0,1,0]：这是一种漫无目的的配置，持有仓位直至被止损触出。
    • [0,0,0]：无任何障碍，仓位永久锁定，不产生任何标签。

图 3.1 展示了三重障碍法的两种不同配置。左图配置为 [1,1,0]，首个被触及的障碍为下方水平障碍；右图配置为 [1,1,1]，首个被触及的障碍为垂直障碍。

3.5 学习交易方向与仓位规模

本节将讨论如何对样本进行标签化，以便 ML 算法能够同时学习下注的交易方向与仓位规模。当我们没有底层模型来确定持仓方向（多头或空头）时，需要通过学习来判断交易方向。在此情况下，我们无法区分止盈障碍与止损障碍，因为这需要预先知晓交易方向。学习交易方向意味着要么不设置水平障碍，要么水平障碍必须对称。代码片段 3.3 实现了函数 getEvents，用于找到首次触及障碍的时间。该函数接受以下参数：

  • close：一个 pandas 价格序列。

  • ptSl：设置两个障碍宽度的非负浮点数。值为 0 表示相应的水平障碍（止盈和/或止损）将被禁用。
  • t1：包含垂直障碍时间戳的 pandas 序列。当需要禁用垂直障碍时，传入 False。

假设 \(I=1E6\) 的均值， \(h=1E3\)，则单一标的需要评估的条件数量多达十亿个。许多 ML 任务在不熟悉多线程的情况下计算开销巨大，本任务便是其中之一。这正是并行计算发挥作用之处。第 20 章讨论了若干 multiprocessing 函数，这些函数将在全书中广泛使用。

3.6 元标签化（meta-labeling）

假设你已有一个用于判断下注方向（做多或做空）的模型，只需学习下注规模，其中包含不下注（规模为零）的可能性。这是从业者经常面临的情形。我们往往已知是否想买入或卖出某产品，剩下的问题仅仅是该笔交易应承担多少资金风险。我们不希望 ML 算法去学习交易方向，只需告知我们合适的仓位规模。此时，你大概不会对以下情况感到惊讶：迄今为止，尚无任何书籍或论文讨论过这一常见问题。所幸，这一缺憾在此终结。我将这一问题称为元标签化（meta-labeling），因为我们希望构建一个次级 ML 模型，用以学习如何利用主外生模型。我们无需重写一个全新的 getEvents 函数，只需对前述代码进行若干调整以支持元标签化。首先，我们接受一个新的可选参数 side（默认值为 None），其中包含由主模型确定的下注方向。当 side 不为 None 时，函数即知元标签化已生效。其次，由于此时已知交易方向，我们可以有效区分止盈与止损。水平障碍无需如第 3.5 节所述那样对称。参数 ptSl 为包含两个非负浮点数的列表，其中 ptSl[0] 为乘以 trgt 以设定上方障碍宽度的系数，ptSl[1] 为乘以 trgt 以设定下方障碍宽度的系数。当任一值为 0 时，相应障碍禁用。代码片段 3.6 实现了上述改进。

3.7 如何使用元标签化（Meta-标签化）

二分类问题存在第一类错误（假阳性）与第II类错误（假阴性）之间的权衡。一般而言，提高二分类器的真阳性率往往会导致其假阳性率随之上升。二分类器的受试者工作特征（ROC）曲线衡量了提高真阳性率所需付出的代价，即接受更高假阳性率的程度。

图 3.2 展示了所谓的"混淆矩阵"。在一组观测样本中，部分样本具有某一特征（正例，左侧矩形），其余样本不具有该特征（负例，右侧矩形）。二分类器预测某些样本具有该特征（椭圆区域），其中TP区域为真阳性，TN区域为真阴性。由此产生两类错误：假阳性（FP）与假阴性（FN）。"精确率"（Precision）是TP区域面积与椭圆总面积之比。"召回率"（Recall）是TP区域面积与左侧矩形面积之比。召回率（又称真阳性率）在分类问题中的含义，类似于假设检验中的"检验功效"。"准确率"（Accuracy）是TP与TN区域面积之和除以全部样本数量（正方形面积）。一般而言，降低FP区域面积往往以增大FN区域面积为代价，因为更高的精确率通常意味着预测正例更少，从而导致召回率降低。尽管如此，精确率与召回率存在某种最优组合，使分类器的整体效率最大化。F1 分数以精确率与召回率的调和平均值衡量分类器效率（详见第 14 章）。

元标签化（meta-labeling）在希望获得更高 F1 分数时尤为有用。首先，构建一个召回率较高的模型，即便精确率并不突出；其次，对主模型预测为正例的样本应用元标签化，以弥补精确率不足的问题。

元标签化（meta-labeling）通过过滤假阳性来提升 F1 分数，此时大多数阳性已由主模型识别。换言之，二级 ML 算法的作用是判断主（外生）模型产生的阳性信号究竟是真阳性还是假阳性。其目的并非主动发现下注机会，而是决定我们应对已呈现的机会采取行动还是放弃。元标签化是一个极为强大的工具，还有以下四个额外原因。第一，ML 算法常被批评为黑箱（见第 1 章）。元标签化允许你在白箱模型（如基于经济理论的基本面模型）之上构建 ML 系统。这种将基本面模型转化为 ML 模型的能力，使元标签化对"量化基本面"机构尤为有用。第二，应用元标签化时，过拟合的影响较为有限，因为 ML 不决定下注方向，只决定仓位规模。第三，通过将方向预测与规模预测解耦，元标签化支持更为复杂的策略结构。例如，驱动上涨的特征与驱动下跌的特征可能各有不同。在这种情况下，你可以基于某一主模型的买入建议，专门为多头仓位开发一套 ML 策略；同时基于另一完全不同的主模型的卖出建议，专门为空头仓位开发另一套 ML 策略。第四，在小仓位上实现高准确率而在大仓位上准确率低终将使你亏损殆尽。识别优质机会与合理调整仓位规模同等重要，因此专门开发一个 ML 算法以正确完成这一关键决策（仓位规模）是完全合理的。我们将在第 10 章重新讨论这第四点。根据我的经验，元标签化 ML 模型能够比标准标签化模型提供更稳健、更可靠的结果。

3.8 量化基本面之道

你可能在新闻报道中看到过，许多对冲基金正在拥抱量化基本面（quantamental）方法。简单的 Google 搜索便可发现，包括一些最传统的对冲基金在内，许多基金正投入数千万美元开发旨在将人类专业知识与量化方法相结合的技术。事实证明，元标签化（meta-labeling）正是这些人一直期待的东西。让我们来看看原因所在。假设你拥有一组你认为能够预测某些价格的特征，只是还不知道如何加以利用。由于你没有模型来确定每次下注的交易方向，你需要同时学习交易方向和仓位规模。你运用第 3.5 节所学的内容，基于具有对称水平障碍的三重障碍法生成一些标签。此时，你已准备好在训练集上拟合算法，并在测试集上评估预测的准确性。或者，你也可以采取以下做法：

  1. 使用主模型的预测结果，生成元标签。请记住，在这种情况下水平障碍不需要是对称的。
  2. 在相同的训练集上再次拟合你的模型，但这次使用你刚刚生成的元标签。
  3. 将第一个 ML 模型的"交易方向"与第二个 ML 模型的"仓位规模"相结合。

你始终可以在任何主模型之上添加一个元标签化层，无论该模型是 ML 算法、计量经济学方程、技术交易规则还是基本面分析等。这也包括由人类纯粹凭借直觉生成的预测。在这种情况下，元标签化将帮助我们判断何时应该采纳或放弃某位主观 PM 的建议。此类元标签化 ML 算法所使用的特征可以涵盖从市场信息、生物统计数据到心理评估的各类指标。例如，元标签化 ML 算法可能发现，当市场出现结构突变（第 17 章）时，主观 PMs 往往能做出尤为准确的判断，因为他们可能更快地察觉到市场状态的转变。反之，该算法也可能发现，处于压力状态下的 PMs——例如睡眠时间减少、身体疲劳、体重变化等——往往会做出不准确的预测。1 许多职业都要求定期进行心理测评，一个 ML 元标签化算法可能会发现，这些测评分数对于评估我们当前对某位 PM 预测的置信程度同样具有参考价值。也许这些因素对主观 PMs 毫无影响，他们的大脑能够独立于情绪运转，如同冷静计算的机器。我的猜测是事实并非如此，因此元标签化应当成为每家主观对冲基金不可或缺的 ML 技术。在不远的将来，每家主观对冲基金都将转型为量化基本面（quantamental）公司，而元标签化为它们提供了一条清晰的转型路径。

3.9 删除不必要的标签

某些 ML 分类器在类别严重不平衡时表现欠佳。在此类情况下，最好删除极为稀有的标签，专注于更常见的结果。代码片段 3.8 给出了一个递归删除与极稀有标签相关观测值的流程。函数 dropLabels 递归剔除那些出现频率低于 minPct 比例的类别所对应的观测值，除非仅剩两个类别。

1 您可能至少知道一家大型对冲基金，该基金每天都会监测其研究分析师的情绪状态。

顺便提及，您可能需要删除不必要标签的另一个原因，是 sklearn 中已知的如下缺陷：https://github.com/scikit-learn/scikit-learn/issues/8566. 此类缺陷源于 sklearn 实现中非常基础的假设，解决起来远非易事。在本例中，错误源于 sklearn 选择使用标准 numpy 数组而非结构化数组或 pandas 对象的设计决策。在您阅读本章时，乃至在不远的将来，该问题都不太可能得到修复。在后续章节中，我们将研究通过构建自定义类、扩展 sklearn 功能来规避此类实现错误的方法。
