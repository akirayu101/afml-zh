第二部分：建模

使用交叉验证调优超参数

9.1 动机

超参数调优是拟合 ML 算法的关键步骤。调优不当时，算法很容易过拟合，实盘表现也会令人失望。因此，ML 文献特别强调：所有调过的超参数都要经过交叉验证。第 7 章已经说明，金融领域的交叉验证（CV）尤其困难，其他领域的常规方案往往会失效。本章将讨论如何用清除式 K 折 CV 调优超参数。参考文献还列出了其他方法，它们可能适合某些特定问题。

9.2 网格搜索交叉验证

网格搜索交叉验证会穷举参数组合，并根据用户指定的评分函数，找出 CV 表现最佳的一组参数。如果我们对数据的底层结构了解不多，这是一个合理的起点。scikit-learn 通过函数 GridSearchCV实现了这套逻辑，该函数可接收 CV 生成器作为参数。出于第 7 章所述原因，我们必须传入自己的 PurgedKFold类（代码清单 7.3），以免 GridSearchCV利用泄漏的信息让 ML 估计器过拟合。

代码清单 9.1 给出了函数 clfHyperFit，它实现了经过清除处理的 GridSearchCV。参数 fit_params可用于传入 sample_weight，而 param_grid包含用于组成参数网格的取值。此外，该函数还允许对调优后的估计器使用 bagging。第 6 章已经说明，对估计器采用 bagging 通常是个好主意；上述函数也加入了相应逻辑。

在元标签应用中，我建议使用 scoring='f1'。原因如下：假设样本里有大量负类，即标签为“0”的样本。如果分类器把所有样本都预测为负类，仍会得到很高的 'accuracy'或 'neg_log_loss'，尽管它根本没有从特征中学会区分不同类别。事实上，这种模型的召回率为 0，精确率没有定义（见第 3 章第 3.7 节）。'f1'评分同时考虑精确率和召回率，可以纠正这种虚高的表现（见第 14 章第 14.8 节）。

对于其他非元标签应用，可以使用 'accuracy'或 'neg_log_loss'，因为我们同样关心所有类别的预测。注意，重新编码类别不会影响 'accuracy'或 'neg_log_loss'，但会影响 'f1'。

这个例子恰好说明 sklearn的 Pipeline 有一项局限：它的 fit方法不接收 sample_weight参数，而是接收一个 fit_params关键字参数。这一缺陷已经在 GitHub 上报告，但修复它需要重写并测试许多功能，可能要等一段时间。在此之前，可以使用代码清单 9.2 中的变通方案。

代码清单 9.2 创建了一个名为 MyPipeline的新类，它继承自 sklearn的 Pipeline类，并重写 fit方法来处理参数 sample_weight，随后调用父类实现。

如果你不熟悉这种扩展类的技巧，可以阅读下面这篇 Stack Overflow 入门文章：http://stackoverflow.com/questions/576169/understanding-python-super-with-init-methods。

9.3 随机搜索交叉验证

如果 ML 算法有大量参数，网格搜索交叉验证（CV）的计算量会变得难以承受。此时可以从各参数的分布中抽样；这种替代方法具有良好的统计性质（Bergstra 等人 [2011, 2012]）。它有两个优点：第一，无论问题维度多高，都能直接控制要搜索的组合数量，相当于设定计算预算；第二，那些对性能影响很小的参数不会像网格搜索 CV 那样显著增加搜索时间。

不必另写一个函数来配合 RandomizedSearchCV，我们可以扩展代码清单 9.1，加入随机搜索选项。代码清单 9.3 给出了一种实现。

9.3.1 对数均匀分布

有些 ML 算法只接受非负超参数，许多常用参数都是如此，例如参数 \(C\) （位于 SVC分类器中）和参数 \(\gamma\) （位于 RBF 核中）。1可以从 0 到某个较大值（比如 100）之间的均匀分布中抽取随机数，但这样预计有 99% 的取值会大于 1。对于响应并非线性的参数，这未必是探索可行区域的最高效方法。例如，SVC对参数 \(C\)从 0.01 增至 1 的敏感程度，可能与参数 \(C\)从 1 增至 100 相当。2因此，参数 \(C\)若服从 \(U[0,100]\)分布，抽样会很低效。在这种情况下，更有效的做法是从一个分布中抽样，使样本取对数后服从均匀分布。我把它称为对数均匀分布。由于没有在文献中找到现成定义，下面给出严格定义。

随机变量 \(x\)服从区间 \(a>0\)到 \(b>a\)上的对数均匀分布，当且仅当 \(\log[x]\sim U[\log[a],\log[b]]\)。该分布的累积分布函数（CDF）为：

\[F[x]=\begin{cases}\dfrac{\log[x]-\log[a]}{\log[b]-\log[a]},& a\le x\le b,\\0,& x<a,\\1,& x>b.\end{cases}\]

由此可得概率密度函数（PDF）：

\[f[x]=\begin{cases}\dfrac{1}{x\log[b/a]},& a\le x\le b,\\0,& x<a,\\0,& x>b.\end{cases}\]

注意，CDF 不随对数底数变化，因为

\[\frac{\log[x/a]}{\log[b/a]}=\frac{\log_c[x/a]}{\log_c[b/a]}\]

对任意底数 \(c\)都成立。因此，该随机变量并不依赖 \(c\)。代码清单 9.4 在 scipy.stats中实现并测试了一个随机变量，其中 \([a,b]=[1E-3,1E3]\)，所以 \(\log[x]\sim U[\log[1E-3],\log[1E3]]\)。图 9.1 展示了样本在对数尺度上的均匀性。

9.4 评分与超参数调优

代码清单 9.1 和 9.3 在元标签应用中设置 scoring='f1'；在其他应用中，则设置 scoring='neg_log_loss'，而不是默认的 scoring='accuracy'。虽然准确率更容易直观理解，但调优投资策略的超参数时，我建议使用 neg_log_loss。下面解释原因。

假设你的 ML 投资策略以很高概率预测应当买入某只证券。仓位会随策略信心提高，因此你会建立较大的多头仓位。如果预测错误，市场反而下跌，你就会损失很多钱。然而，无论错误买入预测的概率高还是低，准确率都把它们同等对待；而且，准确率还可能让一次低置信度的正确预测抵消一次高置信度的错误预测。

投资策略依靠“高置信度地预测正确标签”赚钱。低置信度正确预测带来的收益，不足以弥补高置信度错误预测造成的损失。因此，准确率无法真实衡量分类器表现。相比之下，对数损失3 （又称交叉熵损失）会在真实标签给定时计算分类器的对数似然，从而把预测概率纳入考量。对数损失可估计为：

\[L[Y,P]=-\log[\operatorname{Prob}[Y\mid P]]=-N^{-1}\sum_{n=0}^{N-1}\sum_{k=0}^{K-1}y_{n,k}\log[p_{n,k}].\]

其中：

  • \(p_{n,k}\)表示分类器把第 \(n\)个观测预测为某一标签的概率，该标签为 \(k\)。
  • \(Y\)是一个 one-hot 二元指示矩阵，共有\(K\)个可能标签，并满足 \(y_{n,k}=1\)，当且仅当观测 \(n\)被分配标签 \(k\) （ \(K\)个可能标签之一）；否则为 0。

假设分类器两次都预测为 1，而真实标签分别是 1 和 0。第一次预测正确，第二次预测错误，所以准确率为 50%。图 9.2 绘制了预测概率处于 \([0.5,0.9]\)时的交叉熵损失。可以看到，在图的右侧，高概率错误预测使对数损失很大，尽管所有情况下的准确率都是 50%。

相比准确率，更应使用交叉熵损失还有第二个原因。CV 会结合样本权重为分类器评分（见第 7 章第 7.5 节）。第 4 章讲过，观测权重由观测的绝对收益率决定。这意味着，加权交叉熵损失会用计算盯市损益（PnL）时涉及的变量来评估分类器：正确标签决定交易方向，预测概率决定仓位大小，样本权重代表该观测的收益或结果。对金融应用的超参数调优而言，这才是合适的 ML 性能指标，而不是准确率。

使用对数损失作为评分统计量时，通常会把符号取反，记作 neg_log_loss。这样做只是为了让结果更符合直觉：和准确率一样，负对数损失越高越好。请注意这个 sklearn缺陷，尤其是在使用 neg_log_loss：https://github.com/scikit-learn/scikit-learn/issues/9144。为绕过该缺陷，应使用第 7 章给出的 cvScore函数。
