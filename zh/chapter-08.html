第二部分：建模

特征重要性

8.1 动机

金融研究中最为普遍的错误之一，是将数据输入某个 ML 算法，对预测结果进行回测，然后不断重复这一流程，直到出现一个"好看"的回测结果为止。学术期刊中充斥着此类伪发现，即便是大型对冲基金也常常陷入这一陷阱。即便回测采用的是滚动前推（WF）样本外方式，也无济于事。在同一数据上反复进行检验，极有可能导致虚假发现。这种方法论错误在统计学家中臭名昭著，被视为学术造假，美国统计学会（American Statistical Association）在其职业道德准则中对此明确提出警告（American Statistical Association [2016]，Discussion #4）。在标准显著性水平（假阳性率）5% 下，通常需要约 20 次这样的迭代才能"发现"一个（虚假的）投资策略。本章将探讨为何这种方法是对时间与金钱的浪费，以及特征重要性如何提供一种替代思路。

8.2 特征重要性的重要意义

金融行业有一个令人瞩目的现象：许多经验丰富的投资组合经理（包括大量具有量化背景的从业者）并未意识到对回测进行过拟合是多么容易的一件事。如何正确进行回测并非本章的主题，我们将在第11至15章中详细探讨这一极为重要的议题。本章的目标是阐明在开展任何回测之前必须执行的分析之一。

假设给定一对矩阵 \((X,y)\)，分别包含某一金融工具的特征与标签。我们可以在 \((X,y)\) 上拟合一个分类器，并通过清除式 \(k\)折交叉验证（CV）来评估泛化误差，如第7章所述。假设我们取得了良好的性能，下一个自然而然的问题便是理解哪些特征对该性能有所贡献。或许我们可以添加一些能够增强分类器预测能力所依赖信号的特征，也可以剔除那些仅对系统引入噪声的特征。

值得注意的是，理解特征重要性打开了众所周知的黑盒。如果我们能理解哪些信息来源对分类器不可或缺，便可洞察其所识别的规律。这也是黑盒论调被ML怀疑者过度渲染的原因之一。诚然，算法是在没有我们指导的情况下自主学习的（这正是ML 的全部意义所在！），在黑盒中完成训练，但这并不意味着我们不能（或不应该）审视算法所发现的内容。

一旦找到重要特征，我们可以通过一系列实验深入探究。这些特征是否在任何时候都重要，还是仅在某些特定环境下才发挥作用？随时间推移，是什么触发了重要性的变化？这些机制切换能否被预测？那些重要特征对其他相关金融工具是否同样适用？它们对其他资产类别是否相关？在所有金融工具中，哪些是最相关的特征？在整个投资域中秩相关性最高的特征子集是什么？这远比愚蠢的回测循环更具价值。请允许我将这一格言作为本书最核心的经验之一：

8.3 存在替代效应的特征重要性

我发现根据特征重要性方法是否受替代效应影响来加以区分是有益的。在此语境下，替代效应是指当某一特征的估计重要性因其他相关特征的存在而被削弱的现象。替代效应是统计学与计量经济学文献中所称"多重共线性"的机器学习对应概念。处理线性替代效应的一种方法是对原始特征进行 PCA，然后对正交特征执行特征重要性分析。详见 Belsley 等人 [1980]、Goldberger [1991, pp. 245–253] 以及 Hill 等人 [2001] 的进一步论述。

8.3.1 平均不纯度下降

平均不纯度下降（MDI）是一种快速的解释性重要性（样本内、IS）方法，专用于基于树的分类器，例如 RF。在每棵决策树的每个节点处，所选特征对其接收到的子集进行划分，使得不纯度得以

降低。因此，我们可以从每棵决策树中推导出整体不纯度下降中可归因于各特征的比例。鉴于我们拥有一个由多棵树构成的随机森林，可以对所有基估计器的上述值取平均，并据此对特征进行排序。详见 Louppe 等人 [2013] 有关详细说明，参见相关章节。在使用 MDI 时，有几点重要事项需要注意：

  1. 当某些特征被基于树的分类器系统性地忽略而偏向于其他特征时，就会产生遮蔽效应。为避免此问题，请设置 max_features=int(1) 在使用 sklearn 的 RF 类时。这样，每个节点层级只随机考虑一个特征。
    1. 每个特征都有机会（在某些随机树的某些随机层级上）降低不纯度。
    2. 确保重要性为零的特征不参与平均计算，因为特征重要性为 0 的唯一原因是该特征未被随机选中。请将这些值替换为 np.nan.
  2. 该过程显然存在 IS 问题。每个特征都会具有一定的重要性，即便它们完全不具备任何预测能力。
  3. MDI 无法推广到其他非树型分类器。
  4. 从构造上看，MDI 具有一个良好性质：各特征重要性之和为 1，且每个特征的重要性均介于 0 到 1 之间。
  5. 该方法未能处理相关特征存在时的替代效应。MDI 会因可替换特征的可互换性而稀释其重要性：两个完全相同的特征的重要性各自减半，因为它们被随机选中的概率相等。
  6. Strobl et al. [2007] 通过实验表明，MDI 对某些预测变量存在偏差。White and Liu [1994] 认为，对于单棵决策树而言，这种偏差源于常用不纯度函数对类别数较多的预测变量给予了不公平的优势。

Sklearn 的 RandomForest 类将 MDI 作为默认的特征重要性评分。这一选择很可能是因为 MDI 可在训练过程中即时计算，计算成本极低。1 代码清单 8.2 展示了 MDI 的一种实现，其中涵盖了上述各项注意事项。

8.3.2 平均准确率下降（MDA）

平均准确率下降（MDA）是一种较慢的预测型重要性（样本外，OOS）方法。其步骤如下：首先，拟合一个分类器；其次，根据某种性能评分（准确率、负对数损失等）计算其样本外性能 OOS；第三，逐列对特征矩阵（X）进行置换，每次置换一列，并在每次置换后重新计算性能 OOS。某一特征的重要性由该列置换所导致的性能损失来衡量。若干重要注意事项包括：

  1. 该方法可应用于任意分类器，而不仅限于基于树的分类器。
  2. MDA 并不局限于将准确率作为唯一的性能评分指标。例如，在元标签化（meta-labeling）应用场景中，我们可能更倾向于使用 F1 而非准确率来评估分类器（详见第14章第14.8节的说明）。这也是"排列重要性"（permutation importance）这一名称更具描述性的原因之一。当评分函数不对应某个度量空间时，MDA 的结果应作为排名使用。
  3. 与 MDI 类似，该方法在存在相关特征时同样容易受到替代效应的影响。对于两个完全相同的特征，MDA 始终会将其中一个视为冗余。不幸的是，MDA 会使这两个特征看起来都完全无关紧要，即便它们实际上至关重要。
  4. 与 MDI 不同，MDA 有可能得出所有特征均不重要的结论。这是因为 MDA 基于 OOS 的性能表现。
  5. CV 必须经过清除（purging）和禁运期（embargo）处理，原因详见第7章。

代码清单 8.3 实现了带有样本权重和清除（purging）处理的 MDA 特征重要性计算 \(k\)-折 CV，并以负对数损失或准确率作为评分标准。它将 MDA 重要性衡量为改进量（从打乱特征到不打乱特征）相对于最大可能得分（负对数损失为 0，或准确率为 1）的函数。注意，在某些情况下，改进量可能为负值，这意味着该特征实际上对 ML 算法的预测能力有害。

8.4 排除替代效应的特征重要性

替代效应可能导致我们丢弃恰好具有冗余性的重要特征。在预测场景中，这通常不是问题，但当我们试图理解、改进或简化模型时，可能会导致错误结论。因此，以下单特征重要性方法可以作为 MDI 和 MDA 的有益补充。

8.4.1 单特征重要性

单特征重要性（SFI）是一种横截面预测重要性（样本外）方法。它逐一计算每个特征单独使用时的 OOS 性能得分。以下几点值得注意：

  1. 该方法可应用于任意分类器，而不仅限于基于树的分类器。
  2. SFI 不局限于以准确率作为唯一的性能评分指标。
  3. 与 MDI 和 MDA 不同，由于每次仅考虑单个特征，不会产生替代效应。
  4. 与 MDA 类似，它可能得出所有特征均不重要的结论，因为绩效是通过 OOS CV 来评估的。

SFI 的主要局限在于，使用两个特征的分类器可能优于两个单特征分类器的 bagging 集成。例如：（1）特征 B 可能只在与特征 A 组合时才有用；或（2）即便特征 B 单独使用时精度不高，它仍可能有助于解释特征 A 的分裂。换言之，SFI 无法捕捉联合效应和层次重要性。另一种方案是从特征子集中计算 OOS 绩效得分，但随着特征数量增加，该计算将变得难以处理。代码清单 8.4 展示了 SFI 方法的一种可能实现。关于函数 cvScore 的讨论见第 7 章。

8.4.2 正交特征

如第 8.3 节所述，替代效应会稀释 MDI 所测量的特征重要性，并显著低估 MDA 所测量的特征重要性。一种部分解决方案是在应用 MDI 和 MDA 之前对特征进行正交化。主成分分析（PCA）等正交化方法并不能消除所有替代效应，但至少可以缓解线性替代效应的影响。

考虑一个矩阵 \(\{X_{t,n}\}\) 的平稳特征矩阵，观测值为 \(t=1,\ldots,T\) ，变量为 \(n=1,\ldots,N\)。首先，我们计算标准化特征矩阵 \(Z\)，使得 \(Z_{t,n}=\sigma_n^{-1}(X_{t,n}-\mu_n)\)，其中 \(\mu_n\) 是 \(\{X_{t,n}\}_{t=1,\ldots,T}\) 和 \(\sigma_n\) 是其标准差。 \(\{X_{t,n}\}_{t=1,\ldots,T}\)其次，我们计算特征值 \(\Lambda\) 与特征向量 \(W\) ，使得 \(Z'ZW=W\Lambda\)，其中 \(\Lambda\) 是一个 \(N\times N\) 的对角矩阵，其主对角线元素按降序排列，且 \(W\) 是一个 \(N\times N\) 为正交规范矩阵。第三步，我们将正交特征推导为 \(P=ZW\)。我们可以通过以下方式验证特征的正交性： \(P'P=W'Z'ZW=W'W\Lambda W'W=\Lambda\).

对角化操作是在 \(Z\) 而非 \(X\)上进行的，原因有二：（1）对数据进行中心化处理可确保第一主成分正确指向观测值的主方向，这等价于在线性回归中加入截距项；（2）对数据进行标准化处理使 PCA 着重解释相关性而非方差。若不进行标准化，前几个主成分将由 \(X\) 方差最大的列所主导，从而难以揭示变量间的结构关系。代码清单 8.5 计算了能够解释至少 95% 方差所需的最少正交特征数量。 \(Z\).

除解决替代效应外，使用正交特征还具有两项额外优势：（1）正交化也可用于降低特征矩阵的维度 \(X\)，通过剔除与小特征值相关联的特征。这通常可以加快 ML 算法的收敛速度；（2）分析在专门用于解释数据结构的特征上进行。

我想着重强调后一点。贯穿全书的一个普遍担忧是过拟合风险。ML 算法总能找到某种模式，即便该模式不过是统计上的偶然。对于任何方法所识别出的所谓重要特征，你都应保持怀疑态度，包括 MDI、MDA 和 SFI。现在，假设你使用 PCA 推导正交特征。你的 PCA 分析在完全不借助标签知识的情况下（即无监督学习），已判定某些特征比其他特征更具"主成分"性质。也就是说，PCA 在不存在任何分类意义上过拟合可能的情况下完成了对特征的排序。当你的 MDI、MDA 或 SFI 分析利用标签信息将某些特征选为最重要特征，而这些特征恰好与 PCA 在忽略标签信息时选定的主成分特征一致时，这就构成了验证性证据，表明 ML 算法所识别的模式并非完全源于过拟合。

图 8.1 展示了与某一特征向量相关联的特征值（x 轴）与该特征向量所对应特征的 MDI（y 轴）的散点图。Pearson 相关系数为 0.8491（p 值低于 1E-150），这证明 PCA 在未发生过拟合的情况下正确识别了信息性特征并对其进行了合理排序。

我认为计算特征重要性与其对应特征值（或等价地，其 PCA 倒序排名）之间的加权 Kendall's tau 是十分有用的。该值越接近 1，PCA 排名与特征重要性排名之间的一致性就越强。相较于标准 Kendall's tau，偏好加权版本的理由在于：我们希望优先关注最重要特征之间的排名一致性，而对无关（可能含噪）特征的排名一致性并不十分在意。图 8.1 样本的双曲加权 Kendall's tau 为 0.8206。

8.5 并行化特征重要性与堆叠特征重要性

研究特征重要性至少有两种方法。第一种方法是，对于每只证券 \(i\) 在投资标的池中 \(i=1,\ldots,I\)，我们构建数据集 \((X_i,y_i)\)，并行推导各自的特征重要性。例如，记 \(\lambda_{i,j,k}\) 为特征 \(j\) 在标的 \(i\) 按准则 \(k\)下的重要性。然后，我们可以汇总整个标的池的所有结果，得到综合的 \(\Lambda_{j,k}\) 特征重要性 \(j\) 按准则 \(k\)在多种品种上均具有重要性的特征，更可能与某一潜在规律相关联——尤其是当这些特征重要性在各评判标准上呈现出高秩相关性时。深入研究使这些特征具有预测能力的理论机制或许是有价值的。

该方法的主要优势在于可并行计算，运算速度快。其不足之处在于，由于替代效应，重要特征的排名可能在不同品种间发生互换，导致估计量的方差增大。 \(\lambda_{i,j,k}\)。若我们对 \(\lambda_{i,j,k}\) 在足够大的投资标的池中跨品种取平均，上述不足的影响则相对有限。

第二种替代方法是我称之为"特征堆叠"的方式。其做法是将所有数据集 \(\{(\widetilde X_i,y_i)\}_{i=1,\ldots,I}\) 合并为一个统一的组合数据集 \((X,y)\)，其中 \(\widetilde X_i\) 是以下变量的变换形式 \(X_i\) (例如，在滚动追踪窗口上进行标准化处理）。该变换的目的是确保一定程度的分布同质性， \(\widetilde X_i\sim X\)。在这种方法下，分类器必须同时学习哪些特征在所有标的上更为重要，仿佛整个投资域实际上是单一标的一般。

特征堆叠具有以下优势：（1）分类器将在比并行化（第一种）方法所用数据集大得多的数据集上进行拟合；（2）重要性可直接得出，无需为合并结果设计加权方案；（3）结论更具普适性，受离群值或过拟合的影响更小；（4）由于重要性得分不是跨标的取平均，替代效应不会导致这些得分被压低。

我通常更倾向于特征堆叠，不仅在特征重要性分析时如此，凡是分类器能在一组标的上拟合的场合均如此，包括用于模型预测。这降低了将估计器过拟合到特定标的或小数据集的可能性。堆叠的主要缺点是可能消耗大量内存和计算资源，而这正是对 HPC 技术的扎实掌握大有裨益之处（第 20—22 章）。

图 8.3 表明，MDA 同样表现良好。其结果与 MDI 的结果一致——所有信息性特征和冗余特征的排名均优于噪声特征，唯独 R_6，这可能源于替代效应。MDA 的一个不足之处在于均值的标准差偏高，不过这可以通过增加清除（purging）后的 \(k\)折 CV 的分区数来改善，例如从 10 增加到 100（代价是 \(10\times\) 倍的无并行化计算时间）。

图 8.4 表明，SFI 同样表现尚可；然而，少数重要特征的排名低于噪声特征（I_6, I_2, I_9, I_1, I_3, R_5），这可能是联合效应所致。

标签是多个特征组合的函数，若试图独立预测各特征则会忽略其联合效应。尽管如此，SFI 作为 MDI 和 MDA 的补充仍具有价值，恰恰是因为这两类分析各自受到不同类型问题的影响。
