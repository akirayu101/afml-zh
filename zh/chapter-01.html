前言

金融机器学习：一门独立的学科

1.1 写作缘起

机器学习（ML）正在改变我们生活的方方面面。如今，ML 算法已经能完成不少直到不久前还只能由人类专家完成的任务。对金融业来说，现在正是采用这项颠覆性技术的好时机：未来几代人的投资方式都可能因此改变。本书介绍的，是我在二十年实践中反复验证过、具有可靠科学依据的 ML 工具；我也曾用这些工具，为要求极高的机构投资者管理大规模资金。投资类书籍大致分成两种。一种由缺少实战经验的作者写成，数学十分漂亮，描述的却是一个并不存在的世界。定理在逻辑上成立，不代表它在现实中也成立。另一种来自实务界，讲的是现实观察，却缺少严谨的学术理论；作者往往误用数学工具，做出的模型严重过拟合，一投入实际使用就失效。结果是，学术研究和论文发表脱离了金融市场实践，交易与投资界的许多做法又没有扎实的科学基础。我写这本书的第一个原因，就是想跨过学术界与业界之间这道老生常谈的鸿沟。我在两边都工作过，知道跨过去有多难，也知道困在其中一边有多容易。真正可取的做法，是在两者之间找到平衡。本书不会只因为一种理论数学上漂亮就拥护它，也不会只因为一种办法看起来有效就推荐它。我的目标，是把只能从实践中获得的经验，用严谨的方式整理并传递给读者。

第二个原因，是我希望金融真正发挥有益的作用。这些年来，我在学术期刊和报纸上发表过一些文章，批评金融业目前在社会中扮演的角色。江湖骗子抛出荒唐的臆测，大众媒体又推波助澜，投资者于是被诱导着拿自己的财富去赌博。我相信在不远的将来，ML 会成为金融业的主流，科学方法会减少拍脑袋式的猜测，投资也不再等同于赌博。我希望读者能参与这场变革。第三个原因，是许多投资者低估了把 ML 用于投资有多复杂。正在转向“量化基本面”（quantamental）的主观投资机构尤其容易犯这个错误。我担心他们的高期望最终会落空，不是因为 ML 不管用，而是因为他们用错了方法。未来几年，很多机构会把从学术界或硅谷直接拿来的现成 ML 算法用于投资。我的判断是，他们会亏钱——输给更好的 ML 方案。战胜市场中的群体智慧，远比识别人脸或驾驶汽车困难。我希望本书能帮助你处理那些让金融成为 ML 高难度应用场景的问题，例如回测过拟合。金融 ML 本身就是一门学科：它与常规 ML 有联系，却不能简单混为一谈。本书将逐步说明其中的门道。

1.2 金融机器学习项目经常失败的首要原因

量化金融项目的失败率很高，金融 ML 项目尤其如此。少数成功者能吸引大量资金，并持续为投资者交出出色的业绩，但这种结果十分罕见，原因会在本书中逐一解释。过去二十年里，我见过许多人来了又走，也见过不少公司成立后又关门。根据我的经验，这些失败背后都有同一个关键错误。

1.2.1 西西弗斯模式

主观决策型投资组合经理（PM）的投资决定，并不遵循某套明确的理论或推理规则；如果遵循，他们就属于系统化 PM 了。他们阅读即时新闻和各种分析，但主要还是依靠个人判断或直觉。事后，他们可能用某个故事来解释自己的决定，可任何决定都能找到一套说法。由于没人能完全看清这些押注背后的逻辑，投资公司通常让他们各自独立、分隔工作，以此实现分散化。如果你参加过主观决策型 PM 的会议，大概会发现这类会议常常又长又没有重点。每个人似乎都只盯着某一条孤立的个案信息，却在没有事实和实证证据支撑的情况下，一步跳到很远的结论。这并不是说主观决策型 PM 不会成功；恰恰相反，确实有少数人很成功。问题在于，这类 PM 天生不适合组成团队协同研究。把 50 位主观决策型 PM 放在一起，他们

会不断互相影响，到头来公司付了 50 个人的工资，得到的却只相当于一个人的成果。因此，让他们彼此隔离、尽量少交流，确实有道理。但我见过的量化或 ML 项目，只要照搬这套办法，无一例外都会出问题。董事会的想法通常是：既然这套管理方式对主观决策型 PM 有效，那对量化研究员也照做。于是公司招来 50 名博士，要求每个人在六个月内拿出一套投资策略。结果往往适得其反。每位博士都会急着寻找投资机会，最后不是选中一个在过拟合回测里表现极佳的假阳性，就是退回到常规因子投资。这类策略已经十分拥挤，夏普比率很低，好处只是有学术研究撑腰。无论哪一种结果，都会让投资委员会失望，项目也会被取消。即使 50 人中真有 5 人发现了有效规律，赚到的钱通常也不足以覆盖全部 50 人的成本。最终，这 5 人会去别处寻找与贡献相称的回报。

1.2.2 元策略模式

如果公司要求你独自开发 ML 策略，你从一开始就处于明显劣势。做出一套真正有效的投资策略，和批量做出一百套策略，所需的基础投入几乎一样多，复杂工作包括数据整理与处理、HPC 基础设施、软件开发、特征分析、交易执行模拟、回测等。即使公司为这些环节提供共享服务，你仍像宝马工厂里的一名工人：周围车间全都可以用，但公司却要求你一个人造出整辆车。这周你得是焊接大师，下周要当电工，再下周变成机械工程师，接着还得负责喷漆……你会不断尝试、失败，然后绕一圈又回去焊接。这怎么可能合理？据我所知，成功的量化公司无一例外都采用元策略模式（López de Prado [2014]）。因此，本书是一部供团队而非个人使用的研究手册。你会通过各章学会怎样建立一座“研究工厂”，以及怎样设置流水线上的各个工位。每位量化研究员都应专攻一个具体环节，努力做到这一环节的顶尖水平，同时理解整条流程如何运作。本书给出的就是这座工厂的蓝图：团队可以不靠偶然的好运，而以可预期的速度取得研究发现。Berkeley Lab 和其他美国国家实验室一直用这种方式开展科学研究，例如为元素周期表增加 16 种元素，以及为 MRI 和 PET 扫描奠定基础。1这些发现不能归功于某一个人，因为它们来自团队合作，每个人都贡献了自己的一部分。当然，建立这样的金融实验室需要时间，也需要真正懂行并且做过这件事的人。不过，哪种办法更可能成功？是采用已经得到验证、有组织的协作模式，还是让每位量化研究员都像西西弗斯一样，独自把自己的巨石推上山？

1.3 本书结构

本书把一组彼此牵连的问题拆开，再按清晰的顺序讲解。每一章都默认你已经读过前面的内容。第一部分介绍怎样整理金融数据，让 ML 算法能够正确使用它们。第二部分讲怎样基于这些数据，用 ML 算法开展研究。重点是遵循科学流程，真正做出发现，而不是漫无目的地反复搜索，直到偶然冒出一个很可能是假的“好结果”。第三部分讲怎样回测你的发现，并评估该发现为假的概率。前三部分串起整套流程：从数据分析，到模型研究，再到检验研究发现。掌握这些内容后，第四部分会回到数据，介绍一些从中提取有效特征的新方法。最后，由于其中许多工作都需要大量算力，第五部分将用一组实用的 HPC 方法为全书收尾。

1.3.1 按研发流水线划分

在 16、17 世纪，开采金银相对简单。不到一百年，西班牙珍宝船队就让欧洲流通的贵金属总量增加到原来的四倍。那个时代早已过去。今天，勘探者必须使用复杂的工业方法，才能从成吨的矿土中提取肉眼难辨的贵金属颗粒。但这并不意味着黄金产量跌到了历史低点。恰恰相反，如今矿业每年能提取约 2,500 公吨这种“微观黄金”；而整个 16 世纪，西班牙征服者平均每年带走的黄金只有 1.54 公吨！2肉眼能看到的黄金，只占地球黄金总量中微乎其微的一部分。传说中的黄金国其实一直都在那里……只可惜皮萨罗手里拿的是剑，不是显微镜。寻找投资策略也经历了相似的变化。十年前，个人还比较容易发现“宏观 alpha”，也就是用计量经济学之类的简单数学工具就能找到的明显机会；如今，这种可能性正在迅速接近于零。不论经验多丰富、知识多扎实，今天再由个人去寻找宏观 alpha，胜算都极低。真正剩下的 alpha 已经细小到“微观”尺度，要找到它，必须使用资本密集的工业化方法。和黄金一样，alpha 更细小，并不代表总利润更少。今天可供挖掘的微观 alpha，远比历史上曾经存在的宏观 alpha 丰富。这里仍然有很多钱可以赚，但你需要动用强大的工业级 ML 工具。下面看看现代资产管理公司的研发流水线上有哪些工位。

1.3.1.1 数据治理人员

这个工位负责收集、清洗、建立索引、存储和调整数据，再把数据交给研发流水线。数据可能是表格形式，也可能有层级结构；时间可能对齐，也可能没有对齐；既可能是历史数据，也可能是实时数据流。团队成员需要精通市场微观结构以及 FIX 等数据协议，还要开发相应的数据处理程序，弄清每条数据是在什么背景下产生的。例如，一笔报价被撤销后，是换成了另一个价位的新报价，还是直接撤销、没有替代？每类资产都有自己的细节。债券经常发生置换或被赎回；股票会拆股、反向拆股，还涉及投票权等事项；期货和期权需要展期；外汇则不在一个集中式订单簿中交易。这个工位需要很深的专业知识，已经超出本书范围；本章只会谈到数据治理的少数几个方面。

1.3.1.2 特征分析师

这个工位负责把原始数据变成有用的信号，也就是对金融变量具有一定预测能力的信号。团队成员要熟悉信息论、信号提取与处理、可视化、标签设计、样本加权、分类器和特征重要性等技术。例如，特征分析师可能发现，在以下两件事同时发生时，市场大幅抛售的概率特别高：（1）卖方报价被撤销，随后出现市价卖单；（2）买方报价被撤销，随后在订单簿更深处挂出限价买单。这个发现本身还不是一套投资策略，却可以用于交易执行、流动性风险监控、做市或建立头寸等不同场景。常见误解是把特征分析师当成策略开发者。实际上，他们的工作是收集研究发现、整理成库，供流水线上的多个工位使用。第 2—9 章和第 17—19 章都与这个关键工位有关。

1.3.1.3 策略研究员

这个工位把有用的特征变成真正可以运行的投资算法。策略研究员会查阅特征库，从中寻找开发投资策略的思路。这些特征来自不同分析师对多种金融工具和资产类别的研究。策略研究员要把这些零散观察串起来，提出一套能够统一解释它们的理论。策略本身只是一个实验，用来检验这套理论是否成立。团队成员应当是既懂数据科学，又深入了解金融市场和经济运行的人。要注意，理论必须解释一大批重要特征，尤其要指出一种经济机制：为什么市场中的某类参与者会把钱输给我们？原因是行为偏差、信息不对称，还是监管限制？特征可以由黑箱发现，但策略必须在白箱中构建。把几个已经入库的特征简单拼在一起，算不上理论。策略定稿后，策略研究员会写出包含完整算法的原型代码，交给下面介绍的回测团队。第 10 章和第 16 章主要对应这个工位；当然，不能指望一本书公开具体的投资策略。

1.3.1.4 回测团队

这个工位评估投资策略在不同情景下能否赚钱。其中一个重要情景是：如果历史重演，策略会有什么表现。但真实发生过的历史路径，只是某个随机过程的一种可能结果，并不一定是未来最可能出现的路径。因此，回测还必须根据策略已知的优点和弱点，设计并评估其他情景。团队成员应当是深入掌握实证研究和实验方法的数据科学家。优秀的回测人员还会把策略的产生过程纳入分析，也就是考虑关于策略来源的元信息。尤其是，他们必须统计为了筛出这套策略一共尝试了多少次，并据此评估回测过拟合的概率。出于第 11 章会解释的原因，这项评估结果不会交给流水线上的其他工位重复使用。回测结果只报告给管理层，不与其他人共享。第 11—16 章会介绍这个工位所做的分析。

1.3.1.5 部署团队

部署团队负责把策略代码接入生产系统。有些组件可以由多套策略复用，特别是当这些策略使用共同特征时。团队成员应当是算法专家和功底扎实的数学程序员。他们的一项任务，是确保上线版本在逻辑上与收到的原型完全一致；另一项任务，是充分优化实现，把生产环境中的延迟降到最低。生产计算通常对时间十分敏感，因此这个团队会大量使用进程调度器、自动化服务器（Jenkins）、向量化、多线程、多进程、图形处理器（GPU-NVIDIA）、分布式计算（Hadoop）、高性能计算（Slurm）以及其他并行计算技术。第 20—22 章会讨论与金融 ML 相关、也与这个工位密切相关的若干主题。

1.3.1.6 策略投产后的管理

策略部署后，还要经过一套逐级晋升流程。它包含以下阶段，也就是一套完整的生命周期：

  1. 留出观察期：首先，用回测截止日之后才观察到的数据运行策略。这段数据可能是回测团队事先特意留出的，也可能是开发上线需要时间而自然形成的。如果策略在这段留出期内的表现与回测结果一致，就可以进入下一阶段。
  2. 模拟交易：在这个阶段，策略接入真实的实时数据流运行，但不动用真实资金。这样测得的表现会计入数据解析延迟、计算延迟、执行延迟，以及从观察到信号到建立头寸之间的其他时间损耗。模拟交易要持续到积累了足够证据，能够确认策略表现符合预期为止。
  3. 正式上线：到这一阶段，策略开始管理真实头寸，既可以单独运行，也可以作为策略组合的一部分运行。团队会精确评估它的业绩，包括可归因风险、收益和成本。
  4. 调整配置：策略正式上线后，系统会根据它在生产环境中的表现，在分散化投资组合中频繁、自动地重新评估资金配置。一般来说，策略的资金规模沿一条凹函数曲线变化：刚上线时配置很小；随着时间推移，如果表现符合预期，配置会逐步增加；再往后，策略业绩开始衰减，配置也会逐渐减少。
  5. 下线：所有策略最终都会停用。当一套策略长期低于预期，时间已经足以让我们判断其背后的理论不再得到实证证据支持时，就应将它下线。

通常，更好的做法是发布策略的新变体，让新旧版本并行运行。每个版本都要经历上述生命周期。出于分散化考虑，旧策略会逐渐获得更小的配置；不过，旧版本拥有更长的实盘记录，由此带来的更高可信度也应纳入配置决策。

1.3.2 按策略组成要素划分

许多投资经理相信，致富的秘诀是实现一个极其复杂的 ML 算法。这种想法多半会让他们失望。如果只要写出一个最先进的分类器就能发财，硅谷大多数人早已成了亿万富翁。成功的投资策略由许多因素共同决定。表 1.1 汇总了开发成功策略时需要应对的挑战，以及本书哪些章节会帮助你解决它们。书中还会多次引用我这些年发表的期刊论文。为了避免重复，我通常会直接引导你阅读其中某篇论文，那里对相关主题有更详细的分析。书中引用的论文都可以从我的网站免费下载预印本：www.QuantResearch.org。

1.3.2.1 数据

  • 问题：输入的是垃圾，输出的也只会是垃圾。
  • 解决办法：使用独有、难以人为操纵的数据。如果只有你能使用这份数据，那么它带来的价值就全归你所有。
  • 具体方法：
    • 第 2 章：正确组织数据。
    • 第 3 章：生成真正有信息量的标签。
    • 第 4、5 章：正确建模非 IID 序列。
    • 第 17—19 章：寻找具有预测力的特征。

1.3.2.2 软件

  • 问题：专业任务需要量身定制的工具。
  • 解决办法：开发自己的类。大家都用同一套流行库，就等于更多竞争者在同一口井里取水。
  • 具体方法：
    • 第 2—22 章：本书每一章都会开发自己的函数。面对你的具体问题，你也要参考书中的示例，编写适合自己的函数。

1.3.2.3 硬件

  • 问题： ML 涉及的任务，有些是整个数学领域中计算量最大的。
  • 解决办法：掌握 HPC。如果条件允许，可以与美国国家实验室合作建设超级计算机。
  • 具体方法：
    • 第 20、22 章：学会从多进程架构的角度思考。每次编写程序库时，都要让函数在结构上可以被并行调用。书中有大量示例。
    • 第 21 章：为量子计算机开发算法。

1.3.2.4 数学方法

  • 问题：数学证明可能要花几年、几十年，甚至几百年。没有投资者会等那么久。
  • 解决办法：采用实验数学。面对难以严格求解的问题，不一定要先给出证明，也可以通过实验找到答案。例如，Bailey、Borwein 和 Plouffe [1997] 为 \(\pi\) （圆周率）找到了一种逐位生成算法，却没有先给出证明；此前，人们一直认为这样的数学发现不可能做到。
  • 具体方法：
    • 第 5 章：熟悉能够保留记忆性的数据变换。
    • 第 11—15 章：使用实验方法评估策略价值，这些方法比历史模拟更可靠。
    • 第 16 章：样本内最优的算法，到了样本外可能表现很差。投资成功不存在数学证明，要用实验方法引导研究。
    • 第 17、18 章：检测结构突变，并量化金融序列携带的信息量。
    • 第 20 章：学习分布式计算的任务排队方法，把复杂任务拆开，加快计算。
    • 第 21 章：熟悉离散方法，包括量子计算机使用的方法，用它们处理难以求解的问题。

1.3.2.5 元策略

  • 问题：业余者总想开发某一套神奇策略，以为世上真有一条致富公式。专业人士则研究怎样批量生产策略。真正值钱的不是造出一辆车，而是建成一座汽车工厂。
  • 解决办法：要像经营企业一样思考。你的目标是把研究实验室办成一座工厂，让真正的发现来自有章法的辛勤工作，而不是一时灵感。这正是物理学家 Ernest Lawrence 的理念；他创立了美国第一所国家实验室。
  • 具体方法：
    • 第 7—9 章：建立一套研究流程，在处理金融特征多重共线性的同时，找出能跨资产类别发挥作用的特征。
    • 第 10 章：把多个预测合并成一次下注。
    • 第 16 章：用稳健、样本外表现良好的方法为策略配置资金。

1.3.2.6 过拟合

  • 问题：常规交叉验证方法在金融问题上会失效。受多重检验和选择偏差影响，金融研究中的大多数“发现”其实是假的。
  • 解决办法：
    • 无论做什么，都要反复问自己：我可能在哪些地方过拟合了？对自己的研究保持怀疑，不断要求自己拿出证据，证明这项工作确实创造了价值。
    • 过拟合违背职业伦理。它给出看似诱人的结果，却无法兑现。如果明知如此还故意过拟合，那就是赤裸裸的科研欺诈。许多学者都这样做，并不能让它变得正确；他们并没有拿任何人的财富冒险，甚至连自己的钱也没冒险。
    • 过拟合还会浪费你的时间、资源和机会。更现实的是，行业只会为样本外收益买单。只有真正为投资者创造了可观财富，你才算取得成功。
  • 具体方法：
    • 第 11—15 章：回测有三种基本范式，历史模拟只是其中之一。任何回测都会有一定程度的过拟合，关键是学会量化它究竟有多严重。
    • 第 16 章：学习稳健的资产配置方法，避免为了贴合样本内信号而牺牲样本外表现。

1.3.3 按常见误区划分

ML 优点很多，却不是包治百病的灵药。它越灵活、越强大，用错时的危害也越大。使用不当的 ML 算法，会把统计上的偶然现象误认成稳定规律。金融数据本来就具有信噪比低的特点，两者叠加后，粗心的使用者几乎必然会越来越快地产出虚假发现。本书会揭示 ML 专家把技术用于金融数据时最常犯的一些错误。表 1.2 列出了其中一部分误区，并标明了讲解相应解决办法的章节。

1.4 目标读者

本书介绍专门用于应对金融数据难题的高级 ML 方法。这里的“高级”，不是说内容晦涩难懂，也不是要介绍深度神经网络、循环神经网络或卷积神经网络的最新变体。相反，本书要回答的是那些已有金融 ML 实践经验的资深研究者会认为至关重要的问题。如果你刚接触 ML，也没有使用复杂算法的经验，那么这本书也许暂时还不适合你。只有在实践中遇到过本书讨论的问题，你才容易理解解决它们有什么价值。阅读本书前，不妨先学习近几年出版的几本优秀 ML 入门书；参考文献中列出了一些。本书的主要读者，是拥有扎实 ML 背景的投资专业人士。我希望你能把从书中学到的知识转化为收益，与我们一起推动金融业现代化，并为投资者创造真正的价值。本书也面向那些已在金融以外多个领域成功落地 ML 算法的数据科学家。如果你曾在 Google 工作，用深度神经网络做人脸识别，但把算法用于金融数据时，效果似乎并不

理想，这本书会对你有所帮助。有时，你可能暂时理解不了某些设计背后的金融逻辑（如元标签化（meta-labeling）、三重障碍法和 fracdiff），但请耐心读下去：等你管理投资组合的时间足够长，游戏规则会逐渐清晰，这些章节的用意也会随之明白。

1.5 阅读要求

投资管理是最具跨学科特点的研究领域之一，本书也如实体现了这一点。要读懂各节内容，你需要具备 ML、市场微观结构、投资组合管理、数理金融、统计学、计量经济学、线性代数、凸优化、离散数学、信号处理、信息论、面向对象编程、并行处理和超级计算等方面的实践知识。Python 已成为 ML 事实上的标准语言，因此本书默认你是一名有经验的开发者。你需要熟悉 scikit-learn（sklearn）、pandas、numpy、scipy、multiprocessing、matplotlib 以及其他几个库。

示例代码在调用这些库的函数时，会使用约定俗成的前缀，例如 pandas 用 pd、numpy 用 np、matplotlib 用 mpl，等等。关于每个库都有很多专著，而且其中的细节怎么学都不嫌多。本书还会讨论这些库在实现上的一些问题，包括目前仍未解决、需要留意的缺陷。

1.6 常见问题

ML 算法能在金融领域发挥什么作用？

许多金融业务都要按预先设定的规则作出决策，例如期权定价、算法执行和风险监控。迄今为止，自动化主要发生在这类业务中，它把金融市场变成了交换信息的超高速、高度互联网络。在完成这些任务时，机器要做的只是尽可能快地执行规则。高频交易就是典型例子。关于这一主题，可参见 Easley、López de Prado 和 O'Hara [2013]的详细论述。

金融业的算法化不可阻挡。从 1968 年 6 月 12 日到 12 月 31 日，纽约证券交易所每周三都休市，好让后台部门处理积压的纸面文件。难以想象吧？今天已完全不同，十年后情况还会更好，因为下一轮自动化不再只是照章办事，而是代替人作出判断。人受情绪支配，会有恐惧、希望和各自的利益诉求，本来就不擅长只依据事实决策，遇到利益冲突时尤其如此。在这类情况下，让机器依据从客观、可靠的数据中学到的事实作判断，对投资者更有利。这不仅适用于投资策略开发，也适用于几乎所有金融咨询领域：发放贷款、评定债券、给公司分类、招聘人才、预测盈利、预测通胀，等等。此外，只要程序这样设定，机器就会始终依法行事。如果作出了可疑的决策，投资者可以回查日志，准确了解当时发生了什么。改进一套算法化投资流程，要比改进一套完全依赖人的流程容易得多。

ML 算法为何能在投资中战胜人类？

还记得人们曾经深信，计算机绝不可能在国际象棋上击败人类吗？《危险边缘》（Jeopardy!）呢？扑克呢？围棋呢？数百万年的进化——一种遗传算法——把我们的猿类大脑调校得很适合在三维、自然规律固定的险恶世界里生存。可一旦要在规则天天变化的高维世界中识别细微模式，这种长期调校反而成了负担。ML 算法在 100 维世界里识别模式，并不比在我们熟悉的三维世界里更难。我们看到算法犯低级错误时常会发笑，但别忘了：算法存在的时间，与人类数百万年的进化史相比只是短短一瞬。它们每天都在进步，我们却没有。人类学习得慢，在金融这样瞬息万变的世界里自然处于劣势。

这是否意味着人类投资者已经没有立足之地了？

完全不是。单凭人，人类下棋胜不过计算机；但单凭计算机，也胜不过由计算机辅助的人。主观型投资组合经理（PM）与 ML 算法对赌时处于劣势，但最好的结果也可能来自两者结合。这就是后来所谓的“量化基本面”（quantamental）方式。全书会介绍许多适合量化基本面团队使用的技术，也就是把人的判断（由基本面变量启发）与数学预测结合起来的方法。特别是第 3 章会介绍一种称为元标签化（meta-labeling）的新技术，让你能在主观决策层之上再叠加一层 ML。

金融 ML 与计量经济学有何不同？

计量经济学是把经典统计方法应用于经济和金融序列。它的核心工具是多元线性回归——一项 18 世纪的技术，高斯在 1794 年之前就已经掌握（Stigler [1981]）。标准计量经济学模型并不会学习。很难相信，只靠协方差矩阵求逆这样简单的办法，就能把握 21 世纪金融体系的复杂性。

任何实证科学都必须以观察为基础建立理论。如果研究者用来为这些观察结果建模的统计工具只有线性回归，就无法识别数据的复杂性，最终得到的理论会简单得离谱，毫无用处。我确信，过去 70 年经济学和金融学没有取得实质性进展，计量经济学是主要原因之一（Calkin and López de Prado [2014a, 2014b]）。

几个世纪里，中世纪天文学家不断观察天象，并建立天体力学理论。这些理论从不考虑非圆形轨道，因为那被视为不神圣、不符合上帝的安排。预测误差大得惊人，人们只好设计越来越复杂的理论来解释它们。直到开普勒大胆考虑非圆形（椭圆形）轨道，一个简洁得多的通用模型才突然能够以惊人的精度预测行星位置。如果天文学家始终不考虑非圆形轨道，会怎样？那么……如果经济学家终于开始考虑非线性函数，又会怎样？我们的开普勒在哪里？金融学至今没有自己的《自然哲学的数学原理》（Principia），因为没有开普勒，就不会有牛顿。

金融 ML 方法不是取代理论，而是为理论指路。ML 算法无须预先指定方向，就能在高维空间中学习模式。一旦弄清哪些特征能预测某种现象，我们就可以提出理论解释，再用独立数据集检验。经济学和金融学专业的学生，与其修计量经济学课，不如去学 ML。计量经济学或许足以让你在金融学术界立足（至少目前如此），但要在业界取得成功，就必须掌握 ML。

对于把 ML 算法斥为黑箱的人，你怎么看？

既然你在读这本书，ML 算法对你来说很可能是白箱：它们是透明、定义明确、清清楚楚的模式识别函数。大多数人没有你掌握的知识；在他们眼里，ML 就像魔术师的箱子：“那只兔子从哪儿来的？你这个巫师究竟怎么骗我们的？”人们会怀疑自己不理解的东西。这些偏见源于无知，而苏格拉底给出的解药很简单：教育。况且，我们有些人很喜欢使用自己的大脑，尽管神经科学家至今仍未完全弄清它究竟如何工作——大脑本身也是一个黑箱。

你偶尔会遇到无可救药的卢德分子。内德·卢德（Ned Ludd）是英格兰莱斯特的一名织工，1779 年，他一怒之下砸坏了两台针织机。工业革命兴起后，对机械化怒不可遏的暴民四处破坏，见到机器就砸。纺织工毁掉的工业设备太多，议会不得不立法，把“破坏机器”定为死罪。1811 至 1816 年间，英格兰许多地区公开反叛；英国用于对付卢德分子的兵力，甚至超过了在伊比利亚半岛与拿破仑作战的兵力。卢德运动最终遭到军队残酷镇压。希望反对“黑箱”的运动不要走到那一步。

为什么书中不讨论具体的 ML 算法？

本书不偏向任何一种特定的 ML 算法。无论用卷积神经网络、AdaBoost、RF、SVM 还是其他算法，你都会遇到许多共通问题：数据结构化、标签化、加权、平稳化转换、交叉验证、特征选择、特征重要性、过拟合和回测等。在金融建模中，这些问题都不容易回答，必须专门为金融建模框架开发相应方法。这正是本书的重点。

关于这个主题，你还推荐哪些书？

据我所知，本书是第一本完整、系统讲解金融专用 ML 方法的书：从金融数据结构，到金融序列标签化、样本加权、时间序列差分……最后用整整一编介绍如何正确回测投资策略。诚然，以前也有少数出版物（大多是期刊论文）把标准 ML 应用于金融序列，但本书提供的并不是这种内容。我的目标，是处理金融 ML 建模中特有、又使问题格外棘手的难点。和任何新学科一样，这个领域发展很快；今后出现重大进展时，本书也会更新。如果你希望后续版本讨论某个特定主题，请联系我：mldp@quantresearch.org。我很乐意加入相应章节，并注明提出建议的读者姓名。

有些小节和章节我看不懂，该怎么办？

我的建议是，先阅读每章末尾列出的参考文献。写作时，我不得不假定读者已经熟悉现有文献，否则本书会失去重点。如果读完这些文献后相关内容还是难以理解，很可能是因为它们讨论的是投资专业人士早已熟悉的问题，只是文献中未必提到。

例如，第 2 章会介绍如何有效调整期货价格以处理展期。大多数从业者都知道这个问题，教科书却很少讨论。我也建议你参加我定期举办的研讨会，在演讲结束后直接向我提问。

本书为什么如此关注回测过拟合？

有两个原因。第一，回测过拟合可以说是整个数理金融领域最重要的未决问题，相当于计算机科学中的“P vs NP”。如果有一种精确方法能防止回测过拟合，我们就可以拿着回测结果去银行兑现。回测不再是销售话术，而会几乎等同于现金。对冲基金可以放心地给投资组合经理分配资金；投资者承担的风险会更小，也愿意支付更高的费用；监管机构则能根据可靠的能力与知识证据向对冲基金经理发放执照，不给骗子可乘之机。在我看来，不讨论这个问题的投资书不值得你花时间。一本书大谈 CAPM、APT、资产配置和风险管理等内容，可支撑这些论点的实证结果在筛选时却没有估计虚假发现概率，为什么还要读它？

第二，ML 是研究工具箱里的一件利器，但也确实很危险。回测过拟合在计量经济学分析中已经是个问题，而 ML 的灵活性更让它始终威胁你的研究。金融领域尤其如此，因为我们的数据集更短、信噪比更低，又没有实验室可以在控制所有环境变量的条件下做实验（López de Prado [2015]）。一本不处理这些隐患的 ML 书，可能会给你的职业生涯带来更多伤害，而不是帮助。

本书采用什么数学符号体系？

刚开始写这本书时，我曾想为每个数学变量或函数分配一个全书唯一的符号。如果本书只讲随机最优控制这样的单一主题，这样做很合适。但本书涉及许多数学领域，每个领域都有自己的惯例。如果我不遵循各自的文献规范，读者查阅参考资料反而更困难；这就意味着有些符号必须重复使用。为避免混淆，每章都会随用随解释其中的符号。大部分数学内容都附有示例代码，因此拿不准时，请一律以代码为准。

第 22 章是谁写的？

一种常见看法是，ML 是 IBM、Google、Facebook、Amazon、Netflix、Tesla 等公司发明或完善的一项新奇而迷人的技术。诚然，科技公司已经成为 ML 的重度用户，近几年尤其如此。这些公司赞助了近期一些最受瞩目的 ML 成果（如 Jeopardy! 和围棋程序），可能进一步加深了这种印象。

不过，读者可能会惊讶地发现，事实上，美国国家实验室是应用 ML 历史最久、经验最丰富的研究机构之一。早在 ML 还没成为热门话题时，这些机构就已经开始使用它，并在几十年里凭借 ML 取得了令人惊叹的科学发现。如果预测 Netflix 接下来该向你推荐哪部电影是一件值得做的事，那么理解宇宙膨胀速度、预测全球变暖对哪些海岸线影响最大，或防止全国电网发生灾难性故障，同样值得。这些正是 Berkeley Lab 等机构每天借助 ML 默默而不懈研究的一部分重大问题。

在第 22 章中，Horst Simon 博士和 Kesheng Wu 博士分别从一家美国大型国家实验室副主任和项目负责人的角度出发，讲述涉及大数据、高性能计算和 ML 的大规模科学研究。与传统大学的研究环境不同，国家实验室通过组建跨学科团队、执行精心设计的流程，并明确划分工作和责任来实现科学突破。这种按研发流水线组织研究的模式，近 90 年前诞生于 Berkeley Lab，并启发了第 1.2.2 节和第 1.3.1 节介绍的元策略范式。
